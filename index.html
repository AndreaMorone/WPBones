Welcome to WP Bones.

WPBones is a library that provides a lot of tools and rules in order to write a WordPress plugin and theme.

Requirement

WP Bones has a few system requirements. However, if you are a newbie in composer, npm, laravel and gulp, please, visit​ the related web site before continue.

Installation

First, download the WPKirk boilerplate. You can use either git command line or github for mac application. You can download the develop or release version:

Develop

$ git clone -b develop https://github.com/gfazioli/WPKirk.git <your plugin name>

Master release

$ git clone https://github.com/gfazioli/WPKirk.git <your plugin name>

Then

$ cd <your plugin name>
$ php bones install

Enter you plugin name or namespace

$ php bones

Then you will see

Bones Version 0.1.3

 Usage:

command [options] [arguments]

Once downloaded, follow the instruction in the README file.

Naming Your Application

After installing WPKirk, you may wish to "name" your plugin. By default, the plugin directory is namespaced under WPKirk, and autoloaded by Composer using the PSR-4 autoloading standard. However, you may change the namespace to match the name of your application, which you can easily do via the namespace Bones command.

For example, at beginning, the plugin is named "WPKirk", you could run the following command from the root of your installation:

php bones namespace WPMyAwesomePlugin

Of course, renaming your plugin is not optional.

Bones Console

Bones is also the name of the command-line interface included with WPKirk boilerplate. It provides a number of helpful commands for your use while developing your plugin. You will use bone command-line interface for start your develop. First of all, try the command-line console by run:

$ php bones

Then, you can change the plugin namespace from WPKirk to MyFirstWPBonesPlugin, for example:

$php bones namespace MyFirstWPBonesPlugin

Awesome! You are quite ready to start to ​develop.
